# 语义关系分组实验：O holy night

## 目标

验证在连通域拆分后，用整张透明母版和带 primitive 编号的覆盖图生成关系 JSON，能否防止语义相关但像素不连通的组件在 Sheet 排版中被拆散。

## 输入与参数

- 来源：`81TSoO-uAfL._AC_SL1500_.jpg`
- 生图方式：内置图像生成模型重建纯 `#ff00ff` 母版，再由现有色键链路生成 Alpha。
- 安装尺寸：450 × 675 mm。
- Sheet：381 × 304.8 mm。
- 页边距：5 mm；裁切外扩：1.5 mm；贴纸间距：2 mm。
- 旋转：关闭；输出：300 DPI。

## 组件结果

- primitive：25 个。
- 初始 active group：25 个。
- `p008`、`p005`、`p006` 分别对应 `O`、`holy`、`night`。
- 其余主要场景（教堂、圣诞马槽、三贤士、天使、羔羊、牛头）在重建母版中已经各自连通，无需语义合并。

## 关系 JSON

```json
{
  "members": ["p008", "p005", "p006"],
  "mode": "rigid",
  "preserve_relative_layout": true,
  "cut_mode": "separate",
  "confidence": 0.98
}
```

关系落地后，原来的三个单独 group 被停用，生成 `gm001`。active group 从 25 个变为 23 个。轮廓阶段将 `gm001` 作为一个排版对象，但保留三个不连通的可见/裁切多边形。

## 对照结果

### 未使用语义关系

- 3 页，100% 尺寸。
- `holy` 被排到第 3 页。
- `O` 和 `night` 被排到第 2 页。
- 短语被拆散，业务结果错误。

### 使用语义关系

- 仍为 3 页，100% 尺寸。
- `O holy night` 保持原始相对位置，作为 `gm001` 整体放在第 3 页。
- 没有增加 Sheet 页数。
- 合并 PDF 为 3 页，每页物理尺寸 381 × 304.8 mm。

## 指标对比

| 指标 | 无语义关系 | 使用语义关系 |
|---|---:|---:|
| 页数 | 3 | 3 |
| 缩放 | 100% | 100% |
| 利用率 | 45.70% | 45.70% |
| 紧凑度 | 75.09% | 75.69% |
| 整齐度 | 85.86% | 78.79% |
| 平衡度 | 72.23% | 74.64% |
| 当前总分 | 77.50% | 76.36% |

总分略降，但语义正确性明显提升。这说明“语义关系”必须作为硬约束，不能只作为现有紧凑度/整齐度评分的一部分，否则排版器可能为了提高几何评分再次拆散短语。

## 阶段耗时

| 阶段 | 耗时 |
|---|---:|
| 纯色母版生成 | 约 31.000 s |
| 创建任务 | 0.081 s |
| 色键与 Alpha | 0.811 s |
| 连通域与初始分组 | 1.162 s |
| 基线轮廓 | 2.603 s |
| 基线四候选排版与最终渲染 | 39.238 s |
| 关系 JSON 应用 | 0.091 s |
| 语义分组后轮廓 | 2.248 s |
| 语义分组后四候选排版与最终渲染 | 35.326 s |

关系推理本轮由当前对话视觉模型查看完整母版和编号覆盖图完成，没有发起独立外部视觉 API 请求，因此没有可分离的服务端推理耗时。正式接入时应在视觉接口调用外层使用单独计时器记录。

## 结论

关系 JSON 有效，而且对本例属于必要数据。它没有改善页数，但纠正了 `O holy night` 被拆页的业务错误，同时没有增加材料成本。下一步产品化需要把“整图 + 编号覆盖图 → 关系 JSON → 合并 groups → 失效 geometry/layout”接入正式任务链路。
